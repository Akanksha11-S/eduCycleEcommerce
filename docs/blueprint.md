# **App Name**: EduCycle

## Core Features:

- User Authentication: Secure user registration and login for buyers, sellers, and admins (akanksha123@gmail.com for admin login).
- Product Listings: Sellers can create, manage, and delete product listings with images. Product data persists after browser refresh.
- Product Browsing: Users can browse products with category, price, and condition filters. Display images, titles, prices, seller info, and condition in product cards.
- Shopping Cart & Wishlist: Users can add items to a shopping cart and wishlist that persists between sessions.
- WTB Board: Users can post 'Want to Buy' requests with descriptions and budget. When a user uploads a request it should be retained between the sessions.
- User Dashboards: Separate dashboards for buyers (purchase history) and sellers (listings management, sales analytics).
- Admin Panel: Admins can manage users and view platform reports. Retain user and revenue data. If there is an error in a specific piece of code, the application continues to work and function properly.

## Style Guidelines:

- Primary color: Muted brown (#99582A) for buttons, links, and highlights. It gives warm academic look and works as versatile focal point.
- Background color: Light, warm off-white (#FEFAE0) to make website modern
- Card Background: Slightly darker shade of the background (#FAF3DD).
- Accent color: Soft, muted green (#CCD5AE) for hover effects and secondary highlights, complement the warm brown tones while adding subtle contrast. The green tones represent the theme 'recycle'
- Destructive color: Standard red (#DC2626) for delete actions and error messages.
- Headlines font: 'Playfair', a serif font for titles and headings to convey elegance.
- Body text font: 'Inter', a sans-serif font for paragraphs, labels, and UI text. Its purpose is to improve readablity.
- Use Card components for product listings and dashboard stats. It adds shadow effect on hover.
- Ensure consistent image sizing, format and quality in order to avoid it getting cropped on both main and details page.
- Utilize Dialog components for confirmation actions and Badge components for product and user statuses. Create badges based on conditions new, used or refurbished.
